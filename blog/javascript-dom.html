JavaScript, HTML and CSS

Although you can make a website with HTML only, when JavaScript, HTML and CSS are all used together, this is what really brings websites to life.

What follows is a brief explanation of the role that each of these technologies plays in helping make websites look good, with greater functionality and improved useability.

HTML

HTML is a markup language that is used to publish the content of a webpage (aka 'document') and it is also used to give a document some structure, allowing it to be more easily maintained and interpreted.

CSS

CSS is a design language that provides us with a set of rules, to help control the way the document looks, in terms of its presentation, layout and formatting.

JavaScript

JavaScript is a programming language, which allows us to give function to our websites. You can use it to build logic into websites, making it possible for users to interact with it and to react and respond to those interactions.

An analogy

You may be thinking that this still sounds a little abstract. What does this all mean?!?

A useful way to think about this is to compare it to a person:

  • HTML is like a skeleton, giving a person their shape and form, but as they are still bare bones, they aren't great to look it!
  • CSS provides the skeleton with skin, hair, and stylish clothes to make them look good.
  • JavaScript is the brain, allowing different body parts to move and take actions.

Control flow and Loops

For JavaScript to act as the 'brain', it needs to be able to use logic to make decisions and manipulate things.

In computer programming 'speak', control flow refers to the order in which the various elements of the code (commands, functions, evaluations, etc) are executed.

In JavaScript, the basic flow for following instructions is to start at the top, with the first statement, ending on the bottom line, with the last statement. However, to make the code more useful, usually there needs to be some sort of deviation from this basic order of events, so that the code can evaluate and consider certain conditions and change course depending on whether they are true or false.

I will be discussing two types of code concepts that allow us to do these sort of things and make computer programs more useful. They are:

  • Conditional statements
  • Loops

Conditional statements

These types of statements are used when you need to take different actions, depending on different conditions. The most common form of conditional statements are based on if, else if, else.

Here's an example to illustrate:

You need your website to provide a greeting, based on the time. If is less than 10:00, the greeting should be "Good morning". If the time is not less that 10:00, but is less than 20:00, then the greeting should be "Good day". Otherwise, the greeting should be "Good evening".

The code to do this would look like this:

if (time < 10) {

greeting = "Good morning";

} else if (time < 20) {

greeting = "Good day";

} else {

greeting = "Good evening";

}

Another type of conditional statement is the switch statement, which can be used to select one of many code blocks to be executed.

Loops

A loop helps in executing one or more statements up to a desired number of times. There are several different types of loops, but I will be discussing the for loop here.

It is a 'conditional' loop, which means that it will keep running for as long as a defined condition is true. It is probably the most common type of loop.

The for loop is made up of these three parts, followed by a statement:

  • initialisaton - usually used to initialise a counter variable
  • condition - an expression that is evaluated before each iteration of the loop. If it is true, the statement is executed. If the expression is false, the loop is exited and the code moves to the first statement after the for construct.
  • final expression - to be evaulated at the end of each iteration. Usually involves updating/incrementing the counter variable.

Here's an example to illustrate:

Imagine you are reading a recipe that has 10 ingredients and you want to make sure you have them all. You can do this with a loop.

//Start with the first item in the list, run though each item until you have done all 10

for (let i = 1; i < 10; i++) {

// check whether you have this ingredient. If you don't, then add it to your shopping list.

// then go back to the for statement and increment

}

// Once the loop has finished, execute the first statement outside of the loop ...

DOM

The DOM (Document Object Model) is a Web API that is used to build websites. It is an object representation of an HTML document that allows programs to change the document structure, style, and content of the document/page.

The DOM represents the document as nodes and objects; that way, programming languages can interact with the page. It is mostly used by web developers via the use of JavaScript, but it is not part of JavaScript itself. In fact, implementations of the DOM can be built for any language, e.g. Python.

It is commonly represented as a tree view, like this:

All the items on the tree are known as nodes and nodes have elements, including HTML elements.

Each HTML element has a JavaScript esquivalent that allows you to use js to access and modify the element.

You can set up "event listeners" which are fired to notify code of "interesting changes" that may affect code execution (such as a mouse click, key press, button click, etc).

We can then use commands such as querySelector or querySelectorAll to get access to what we need from the DOM (e.g. when a button has been clicked) and then potentially modify an item.

Arrays and Objects

Both arrays and objects are used to store collections of data. 'Objects' are more obvious for what they are. They might represent a physical 'thing' or something you use in your daily life, such as a car.

A car object might have properties such as make, model, colour. These properties are known as keys. These keys have values, e.g. make="Toyota", model="Corolla", colour="Blue". You can access data from an object by using key value pairs, e.g: car.make = "Toyota"

An array can be thought of as being an ordered collection, or list. The elements of an array are numbered with an index, starting with zero.

To refer to an item within an array, you need to refer to it by its index. e.g. arrayName[0] is referring to the first item in the array

It is common to use loops to run through all the items in an array, by incremeting the index. You can get the size of the array by using the length property.

e.g. arrayName.length

Functions

A function is a block of code that is designed to perform a particular task. It is executed when something else invokes it. You can give it a name and execute it as many times as you want, just by calling it by its name, e.g. calculateAreaOfCircle()

Functions can have one or more parameters, which will be supplied by the calling code and used inside the function.

They are useful because they can make code easier to read by reducing complexity, and they enable code to be reused.
